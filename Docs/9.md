## 3. Understanding the 'D' (Delayed / Daily) items

Any task or mention that has `delayed: true` in its data will **not appear immediately**.
It drips in after `delayMs` milliseconds. This simulates items that accumulate gradually
over the course of the day.

To mark a real task or mention as delayed, add the suffix `(D)` to the source data,
or set `delayed: true` and a `delayMs` value (e.g. `3000` = 3 seconds for demo,
replace with a longer interval like `60000 * 60` for hourly drip in production).

## 4. Wiring to your real data

The component ships with demo data. To connect it to your backend:

### Contributions / Tasks
Replace `INITIAL_TASKS` with a fetch from your API:

```ts
// Example — fetch user tasks
const res = await fetch(`${process.env.NEXT_PUBLIC_API_URL}/api/v1/user/tasks`);
const tasks = await res.json();
```

### Mentions
Replace `INITIAL_MENTIONS` with a fetch from the Telegram ingestion or a dedicated
mentions endpoint:

```ts
const res = await fetch(`${process.env.NEXT_PUBLIC_API_URL}/api/v1/user/mentions`);
const mentions = await res.json();
// Map: timestamp should be a Date object → new Date(mention.created_at)
```

You can use **TanStack Query** (already in the project) to poll for new mentions:

```tsx
import { useQuery } from "@tanstack/react-query";

const { data: mentions } = useQuery({
  queryKey: ["mentions"],
  queryFn: () => fetch(".../mentions").then(r => r.json()),
  refetchInterval: 15_000, // poll every 15 s
});
```

## 5. Customise the profile card

In `user-profile-section.tsx`, search for the hardcoded values and replace:

| Hardcoded | Replace with |
|-----------|-------------|
| `"SC"` (initials) | Your auth user's initials |
| `"Soumya C."` | `session?.user?.name` from NextAuth |
| `"SRE · NexusOps Team"` | User role from your DB |
| `"14"` (incidents) | Count from `/api/v1/nexus/dashboard` |
| `"7"` (PRs merged) | Count from GitHub Octokit or your DB |
